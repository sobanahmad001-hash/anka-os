# Anka OS — Phase 2 Admin Environment

## Admin Identity
Admin is the main organizational spine of Anka OS.

## Admin Purpose
Admin gives leadership and operations visibility, control, coordination, and oversight across the organization.

## Admin Owns
- organization structure
- departments
- teams
- users
- roles / permissions
- project portfolio visibility
- blocker oversight
- decision oversight
- rules / SOP oversight
- workload visibility
- cross-department coordination

## Admin Primary Questions
1. What projects are active?
2. What is blocked or at risk?
3. Which departments need attention?
4. What decisions matter right now?
5. Where is visibility weak?
6. What should leadership focus on next?

## Admin Core Views
1. Overview
2. Portfolio
3. Blockers
4. Decisions
5. Teams & Allocation
6. Rules & Operating Structure

## Admin Should Not Be
- a generic app collection
- only a settings area
- a workspace shell
- a random dashboard without operating logic

## Admin AI Role
Admin AI is a strategic oversight helper.

It helps with:
- organization summaries
- risk visibility
- repeated blocker detection
- department bottleneck detection
- leadership summaries
- priority guidance

# Test and Deploy Sequence

## After Batch 1
- run app locally
- verify shell/nav/routes
- verify no broken global imports

## After Batch 2
- verify Admin routes
- verify user management still works
- verify overview structure is stable

## After Batch 3
- verify Development routes
- verify Kanban/API docs/terminal positioning
- verify project/dev context is preserved

## After Batch 4
- verify Design and Marketing environments
- verify route clarity
- verify environment separation

## After Batch 5
- verify AI behavior differs by environment
- verify prompts stay context-aware
- verify no generic responses

## Before Push
- git status
- local run
- focused smoke test

## Before Deployment
- clean commit history on branch
- push branch
- test preview deployment first if possible

# Anka OS — Phase 1 Shared Core

## Product Identity
Anka OS is one organizational operating system with Admin as the organizational spine and specialized department environments built on a shared execution core.

## Core Principle
Project management is primary.
AI is a major pillar, but acts as a helper inside execution and coordination.

## Shared Core Entities

### Organizational Spine
- organizations
- departments
- teams
- profiles
- roles_permissions

### Execution Spine
- projects
- project_members
- tasks
- task_dependencies
- project_updates
- blockers
- decisions
- documents

### Intelligence Spine
- ai_project_memory
- ai_session_memory
- activity_events
- rules

## Admin Owns
- org structure
- permissions
- project portfolio visibility
- blocker visibility
- decision visibility
- rules / SOPs
- workload and oversight

## All Environments Inherit
- project
- task
- blocker
- decision
- document
- update
- AI memory
- permissions

## Keep
- profiles
- tasks
- projects
- department and role logic
- task assignment logic
- project progress logic
- admin visibility concepts

## Convert
- app-first structure -> project-first structure
- workspace/window mindset -> environment-based OS
- generic shared apps -> supporting modules under project execution

## Remove / Downgrade
- window-based model as product identity
- broad third-party-style app shelf as core experience
- treating notes/wiki/templates/snippets as equal to project core

## Phase 1 Deliverables
1. Shared core entity map
2. Keep / convert / remove decisions
3. Schema adjustment plan
4. Admin spine definition
5. Environment inheritance rules

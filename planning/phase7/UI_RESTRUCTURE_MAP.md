# UI Restructure Map

## Keep
- core layout foundation
- role/access related screens
- project/task-related screens
- current dashboard foundations where reusable
- environment-relevant modules

## Convert
- generic dashboards -> environment dashboards
- app-first sidebar -> environment-first sidebar
- kanban -> execution queue / workflow-specific board
- user management -> Admin / Teams & Allocation
- API docs -> Development / Code & Context
- dev dashboard -> Development / Overview

## Move
- design-related utilities under Design environment
- marketing-related utilities under Marketing environment
- technical utilities under Development environment
- org visibility tools under Admin

## Downgrade
- generic utility apps as first-class navigation items
- workspace/window identity
- third-party app feel

## Remove Later
- anything that survives only because of old workspace/app logic
- duplicate utilities with weak project linkage
